# Contributing to Agentic RAG Assistant

Thank you for your interest in contributing! Here's how you can help.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/yourusername/agentic-rag-assistant.git
   cd agentic-rag-assistant
   ```
3. **Create a virtual environment**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # or venv\Scripts\activate on Windows
   ```
4. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

## Development Workflow

1. **Create a feature branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** and test thoroughly

3. **Commit your work**:
   ```bash
   git commit -m "Add brief description of changes"
   ```

4. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

5. **Open a Pull Request** with a clear description of changes

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing

- Test your changes manually before submitting
- Ensure the app runs without errors
- Test with different PDF documents
- Verify both dark and light themes work

## Reporting Bugs

- Use the GitHub Issues tab
- Describe the bug clearly
- Include steps to reproduce
- Mention your environment (OS, Python version, etc.)

## Feature Requests

- Create an issue with the label "enhancement"
- Describe the feature and its benefits
- Provide example use cases

## Pull Request Guidelines

- Keep PR focused on a single feature or fix
- Write a clear PR description
- Link related issues
- Ensure all tests pass
- Update documentation if needed

Thank you for contributing! 🎉
